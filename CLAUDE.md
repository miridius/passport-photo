## Dev Server

User views the app at `http://mini:<port>` (not localhost). Always share `http://mini:<port>` when the user needs to test visually.

## Testing Rules

Every behavioral change must have automated tests. No exceptions.

- If a bug is found, write a failing test first, then fix it
- If a feature is added, tests prove it works
- "It builds" is not a test. "It works when I click it" is not a test.
