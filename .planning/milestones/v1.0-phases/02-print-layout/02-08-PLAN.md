    Three CSS/layout fixes in CropEditor.svelte:

    **Fix 1 — Ruler tick border compensation:**

    The crop frame has `border: 1px solid`. The rulers are in adjacent grid cells aligned by row/column size, but their 0%..100% positioning spans the frame's outer box (including border). The frame's content area starts 1px inside. Ruler ticks at 0% and 100% are 1px off from content edges.

    Solution: Add a 1px offset to the ruler containers so their positioning coordinate space matches the frame's content area. On the vertical ruler (`.ruler`), add `margin-top: 1px` and `margin-bottom: 1px` (or use `top: 1px; bottom: 1px` with `position: relative`) to shrink the ruler's effective height by 2px (matching the frame's top + bottom 1px borders). For the horizontal ruler (`.ruler-h`), add `margin-left: 1px` and `margin-right: 1px` similarly.

    Specifically:
    - `.ruler` — add `box-sizing: border-box; padding-top: 1px; padding-bottom: 1px;` so ticks at 0% and 100% align with the frame content edge, not the border edge. Since ticks are `position: absolute` with `top: X%`, the percentage is computed against the padding box. Adding 1px padding on each end shifts the 0%/100% positions inward by 1px.

    Actually, a cleaner approach: The ruler height is determined by the grid (matches the crop-frame row). It includes the frame's 2px of border. To compensate, make the ruler `position: relative` and give the ticks a coordinate space that excludes 1px on each end. Do this by wrapping tick positioning:
    - Change `.ruler` to have `padding-top: 1px; padding-bottom: 1px; box-sizing: border-box;`
    - Since `.tick` uses `position: absolute; top: X%`, and percentage-based positioning on absolutely-positioned children resolves against the padding box — this won't work directly.

    Better approach: Use `calc()` in the tick `top` style to offset by 1px:
    - Vertical ticks: change `style="top: {(mm / H) * 100}%;"` to `style="top: calc({(mm / H) * 100}% + 1px - {(mm / H) * 2}px);"` — this maps 0% to +1px and 100% to -1px, linearly interpolated.
    - Simplified: `style="top: calc(1px + {(mm / H)} * (100% - 2px));"` — this linearly maps 0..H to 1px..(height-1px), exactly matching the content area.
    - Same for horizontal ticks: `style="left: calc(1px + {(mm / W)} * (100% - 2px));"`

    **Fix 2 — Remove gap between frame and horizontal ruler:**

    Change `.crop-workspace` `row-gap: 0.5rem` to `row-gap: 0`. The 0.5rem creates unwanted space between the crop frame and horizontal ruler. The kb-hint can get its spacing via `margin-top` instead.

    Add `margin-top: 0.5rem` to `.kb-hint` to preserve spacing below the ruler.

    **Fix 3 — Dimension brackets closer with connecting lines:**

    Move `.dim-inner` from `left: 0.375rem` to `left: 0.125rem` (closer to frame).
    Move `.dim-outer` from `left: 1.875rem` to `left: 1.25rem` (closer to frame).
    Reduce `.dims` width from `3.5rem` to `2.75rem`.

    Add horizontal connecting lines from bracket endpoints to the frame edge. Each `.dim` already has a `.dim-bracket` with `::before` and `::after` for the horizontal caps. Extend these caps leftward to reach the frame edge:
    - Change `.dim-bracket::before` and `::after` to have `left: -100%` and `width: calc(100% + 7px)` — or more precisely, add a new pseudo-element or adjust the existing ones.

    Simpler approach: Add a `border-left` on each `.dim` element itself (1px line connecting to frame) — but this would be a solid vertical line, not what we want.

    Actually, the user wants connecting lines from the bracket horizontal caps to the frame edge. Currently the caps are 7px wide horizontal lines at the top and bottom of the bracket. Extend these to be longer, reaching from the dim bracket position all the way to the left edge of the `.dims` column (which abuts the frame).

    Change `.dim-bracket::before` and `::after`:
    - `left: calc(-1 * (var(--dim-left) + 3px))` where `--dim-left` is the dim's left offset
    - Actually, simpler: set `right: 50%` and make `width` extend from the bracket center all the way to `left: 0` of the `.dim` parent, then beyond to the `.dims` container left edge.

    Simplest approach: on `.dim-bracket::before` and `::after`, change `left: -3px; width: 7px;` to `right: 50%; width: calc(50% + var(--dim-left));` where `--dim-left` is set as an inline style or CSS variable on `.dim-inner` and `.dim-outer`. The line extends from bracket center to the left edge of `.dims`.

    - On `.dim-outer`, add `--connect-width` CSS variable equal to its `left` value
    - Change `.dim-bracket::before` and `::after` from `left: -3px; width: 7px;` to `left: calc(-1 * var(--connect-width) - 3px); width: calc(var(--connect-width) + 7px);`
    - Set `--connect-width: 0.125rem` on `.dim-inner` and `--connect-width: 1.25rem` on `.dim-outer`

    This makes the bracket caps extend from the dim bracket all the way to the left edge of `.dims` (which is the frame's right border), creating visible connecting lines.
