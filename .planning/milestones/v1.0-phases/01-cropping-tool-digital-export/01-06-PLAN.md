Six changes across CropEditor.svelte and +page.svelte:

1. **Fix arrow key directions (gap #2, major).** The `onPan` function converts screen-space deltas to normalized offsets with a negation (`-dx`). So passing positive dx means "move view left." Currently ArrowLeft passes `onPan(panStep, 0)` which moves view RIGHT. Swap all four:
   - ArrowLeft: `onPan(-panStep, 0)` (was `panStep`)
   - ArrowRight: `onPan(panStep, 0)` (was `-panStep`)
   - ArrowUp: `onPan(0, -panStep)` (was `panStep`)
   - ArrowDown: `onPan(0, panStep)` (was `-panStep`)

2. **Add +/- zoom to kb-hint (gap #3).** Change the kb-hint text to:
   `Drag to pan · Scroll to zoom · Arrows to pan · +/- to zoom · Shift for coarse`

3. **Show 0mm and 45mm ruler labels with units (gap #5).** Change the vertical ruler tick label condition from `mm % 5 === 0 && mm > 0 && mm < H` to `mm % 5 === 0`. For the label text, show `{mm}mm` when `mm === 0 || mm === H`, otherwise just `{mm}`. This adds 'mm' units to the endpoints only.

4. **Add horizontal ruler below the crop frame (UI-04).** Add a new `<div class="ruler-h">` element after `</div><!-- end crop-assembly -->` and before the kb-hint. This ruler runs horizontally from 0mm to 35mm (AU_PASSPORT_SPEC.photoWidthMm). Use the same tick styling approach as the vertical ruler:
   - Create `const W = AU_PASSPORT_SPEC.photoWidthMm;` and `const hTicks = Array.from({ length: W + 1 }, (_, i) => i);`
   - Each tick positioned with `left: {(mm / W) * 100}%`
   - Major ticks every 5mm, labels at 5mm intervals
   - Show `{mm}mm` for 0 and 35, bare `{mm}` for interior labels
   - CSS: height ~1.25rem, position relative, left margin matching the vertical ruler width (2rem) so it aligns with the crop frame left edge. Right margin matching the dims width (3.5rem). Add `border-top: 1px solid rgba(255,255,255,0.08)` to match the vertical ruler's border style.
   - Tick marks: vertical lines extending upward from the bottom edge, 5px for minor, 10px for major.
   - Add `aria-hidden="true"`.
   - Hide on mobile: add `.ruler-h` to the `@media (max-width: 480px)` block with `display: none`.

5. **Keep crop frame at 80vh (per CONTEXT.md).** Do NOT change max-height. The crop frame stays at `max-height: 80vh; width: calc(80vh * 35 / 45);`.

6. **Move export buttons above crop editor (gap #8, per CONTEXT.md).** In `src/routes/+page.svelte`, swap the order of ExportButtons and CropEditor inside `.editor-column`:
   ```svelte
   <div class="editor-column">

   ```
   This ensures buttons are visible at the top without scrolling, and the crop frame at 80vh extends below. Per CONTEXT.md locked decision: "Move export buttons above the crop editor — they must be visible without scrolling. Crop frame stays at current max-height (80vh)."
