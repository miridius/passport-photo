   - Outer wrapper: full-width container with centered crop frame

   - Inside crop frame: `<img>` element bound to `imageState.url` with:
     - `style="transform: translate({panXpx}px, {panYpx}px) scale({scaleFactor}); transform-origin: 0 0;"`
     - `draggable="false"`, `user-select: none`

     - The crop frame has a known CSS width (use `bind:clientWidth` on the frame element)
     - `scaleFactor = clientWidth / (cropState.zoomFraction * imageState.naturalWidth)`
     - `panXpx = -cropState.offsetX * imageState.naturalWidth * scaleFactor`
     - `panYpx = -cropState.offsetY * imageState.naturalHeight * scaleFactor`
   - Semi-transparent overlay: Use a full-page overlay with a transparent cutout over the crop frame. Implementation: the crop frame itself clips the image (overflow:hidden). Outside the frame, use the page background or a sibling overlay div with `pointer-events: none` and a semi-transparent background. Simplest approach: set a dark semi-transparent background on the page area surrounding the crop frame.

   - Conditionally render: if `imageState.element` is null, show PhotoLoader; else show CropEditor
   - Import imageState from crop.svelte.ts
   - Add `touch-action: manipulation` on the body/page level
   - Basic layout: centered, max-width for desktop, full-width for mobile

<task type="auto" tdd="true">
  <name>Task 2: Pan/zoom action and guide overlay with crown hint</name>
  <files>
    src/lib/actions/panZoom.ts,

  <behavior>
    - panZoom action: single pointer drag calls onPan with dx/dy deltas
    - panZoom action: two-pointer pinch calls onZoom with scale factor and center
    - panZoom action: wheel event calls onZoom with scroll-derived factor
    - panZoom.test.ts: test pure gesture math functions (delta calculation, pinch distance ratio)
    - GuideOverlay: chin band positioned at chinGuide fractions of frame height
    - GuideOverlay: crown band positioned at crownGuide fractions of frame height
    - GuideOverlay: crown hint text/annotation visible near crown band
  </behavior>
  <action>
1. Create `src/lib/actions/panZoom.ts`:
   - Export a Svelte action: `export function panZoom(node: HTMLElement, callbacks: { onPan, onZoom })`
   - Set `touch-action: none` on the node (critical for mobile — prevents browser zoom/scroll on crop area)
   - Track active pointers in `Map<number, {x, y}>`
   - `pointerdown`: setPointerCapture, add to map
   - `pointermove`: if 1 pointer → call `callbacks.onPan(dx, dy)` with pixel deltas. If 2 pointers → calculate distance ratio between current and previous, call `callbacks.onZoom(ratio, centerX, centerY)` where center is midpoint of the two pointers relative to the node
   - `pointerup`/`pointercancel`: remove from map
   - `wheel`: call `callbacks.onZoom(e.deltaY > 0 ? 1.1 : 0.9, e.offsetX, e.offsetY)`. Use `e.preventDefault()` to prevent page scroll.
   - Return `{ destroy() }` that removes all listeners
   - Also export pure helper functions for testing: `calculatePinchRatio(prev, curr)` and `wheelToZoomFactor(deltaY)`

2. Wire panZoom into CropEditor.svelte:
   - Add `use:panZoom={{ onPan, onZoom }}` to the crop frame div
   - `onPan(dx, dy)`: convert pixel deltas to normalized deltas (divide by image natural dimensions * scaleFactor), then update cropState using applyPan
   - `onZoom(factor, cx, cy)`: convert center to normalized coords, update cropState using applyZoom

3. Create `src/lib/components/GuideOverlay.svelte`:
   - Import chinGuide, crownGuide from spec.ts
   - Absolutely positioned within the crop frame (position: absolute, inset: 0, pointer-events: none)
   - Chin band: div positioned at `top: {chinGuide.topFrac * 100}%` with height `{(chinGuide.bottomFrac - chinGuide.topFrac) * 100}%`, semi-transparent colored band (e.g., rgba blue/green, ~0.2 opacity)
   - Crown band: div positioned at `top: {crownGuide.topFrac * 100}%` with height `{(crownGuide.bottomFrac - crownGuide.topFrac) * 100}%`, semi-transparent colored band
   - Labels: small text labels "chin zone" and "crown zone" inside or beside the bands
   - Crown hint: a text annotation near the crown band stating "Top of skull, not top of hair" or an icon+tooltip. Keep it visible but unobtrusive — small italic text or a dotted-underline label. Use a distinct visual treatment (e.g., dashed border on crown band, or a small info icon with the hint text).

4. Add GuideOverlay inside CropEditor's crop frame div (after the image, before the close of the frame).

5. Write `src/lib/actions/panZoom.test.ts`:
   - Test `wheelToZoomFactor`: positive deltaY returns >1 (zoom out), negative returns <1 (zoom in)
   - Test `calculatePinchRatio`: two pairs of pointer positions, verify distance ratio
   - Test edge cases: zero distance, same pointer position

IMPORTANT: The crop frame div must have `touch-action: none` via the panZoom action. The surrounding page should have `touch-action: manipulation`. This is critical for COMPAT-02 (Android Chrome).
