Two changes:

1. **Implement binary search quality auto-adjust (gap #9, per CONTEXT.md locked decision).** Replace the current `downloadDigital()` function in ExportButtons.svelte. The new logic:

   a. Compute target dimensions (same as now: `Math.min(srcW, maxW)`, aspect ratio 45/35).
   b. First attempt: call `exportCrop` with `quality: 1.0` (from `AU_PASSPORT_SPEC.digitalExport.quality`).
   c. Check blob size against `AU_PASSPORT_SPEC.digitalExport.minFileSize` (70KB) and `AU_PASSPORT_SPEC.digitalExport.maxFileSize` (3.5MB).
   d. **If blob < 70KB at quality 1.0:** The source image is too small/low-quality. Block the export â€” do NOT trigger download. Instead, set an error state (use `$state`) and display a message: "Source image too small for digital submission (minimum 70KB required). Try a higher-resolution photo." Clear error on next attempt.
   e. **If blob > 3.5MB:** Run binary search on quality. Start with `lo = 0.5, hi = 1.0`. For up to 8 iterations: `mid = (lo + hi) / 2`, call `exportCrop` with `quality: mid`. If blob.size > maxFileSize, set `hi = mid`. Else set `lo = mid`. After the loop, use the last blob that was <= maxFileSize. If no iteration produced a valid size, block export with error: "Could not compress to under 3.5MB. Try cropping tighter."
   f. **If 70KB <= blob <= 3.5MB:** Download directly (already valid).

   Import `exportCrop` and `triggerDownload` (already imported). The `handleExport` helper won't work for this since we need the loop â€” call `exportCrop` directly in `downloadDigital`.

   Add an `exportError` reactive state: `let exportError = $state('');`. Display it below the buttons as a `<p class="export-error">` with red text when non-empty. Clear it at the start of each export attempt.
