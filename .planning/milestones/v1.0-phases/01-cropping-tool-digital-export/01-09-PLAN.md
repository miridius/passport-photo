  <action>
    The `.ruler-h` element sits outside `.crop-assembly` and has no intrinsic width. All its children use `position: absolute` with percentage-based `left`, which computes against the parent's width (0px = nothing renders).

    Fix: Move `.ruler-h` inside `.crop-assembly` as a new row, or keep it outside but give it an explicit width that matches the crop frame.

    Preferred approach: Keep `.ruler-h` outside `.crop-assembly` (the assembly is a flex row for ruler|frame|dims) but bind its width to `frameWidth` (already tracked via `bind:clientWidth`). Set an explicit `width` style on `.ruler-h`:

    ```html
    <div class="ruler-h" style="width: {frameWidth}px; margin-left: 2rem;" aria-hidden="true">
    ```

    This gives `.ruler-h` a concrete pixel width so the percentage-based `left` values on `.htick` children resolve correctly. Keep `margin-left: 2rem` to align with the crop frame (past the vertical ruler). Remove `margin-right: 3.5rem` from the CSS since explicit width handles sizing. The explicit `width` style attribute overrides any CSS width.
